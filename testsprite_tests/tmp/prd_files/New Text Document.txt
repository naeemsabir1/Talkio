3.3 Share Functionality (Critical Test)
WARNING

Requires Physical Device Share intents from real apps (Instagram/YouTube/TikTok) cannot be tested on emulators

Test Scenario A: Cold Start - Instagram Share

Close app completely (swipe from recent apps)
Open Instagram app
Find any post/reel
Tap Share → Select "auraly_clone"
Language sheet should appear
Select "English"
Processing screen displays (3 seconds)
Memo detail screen opens
Press back
Verify memo appears in home screen with Instagram icon
Test Scenario B: Warm Start - YouTube Share

App running in background
Open YouTube app
Find any video
Tap Share → Select "auraly_clone"
Same flow as above
Verify YouTube icon displays
Test Scenario C: Multiple Rapid Shares

Share 3 different links rapidly (Instagram → TikTok → YouTube)
Complete each flow
Verify all 3 memos appear in list
Verify newest memo at top
Verify correct platform icons
Test Scenario D: Invalid URL Handling

Share from unsupported app (e.g., Twitter)
Verify graceful handling (should still work with mock data)

Phase 4: Performance Testing
4.1 App Launch Time
Measure cold start time (app closed → home screen)
Measure warm start time (app backgrounded → resumed)
Target: Cold start < 3 seconds
4.2 Memory Usage
Monitor memory during typical usage
Check for memory leaks during extended use
Target: < 150MB for typical session
4.3 Frame Rate Analysis
Test scrolling in memo list (should maintain 60fps)
Test flashcard animations (should be smooth)
Test audio player scrubbing (should be responsive)
4.4 Image Loading
Test thumbnail loading in memo list
Test full images in memo detail
Target: Thumbnails < 500ms, Full images < 1 second
